# Recette – Parcours critiques COYA.PRO

À exécuter après application des migrations et déploiement.

## Prérequis

- Migrations appliquées sur le projet Supabase (voir [scripts/README-MIGRATIONS.md](../scripts/README-MIGRATIONS.md))
- Application démarrée et accessible

## Parcours 1 : Login → Dashboard

1. Se connecter avec un compte valide
2. Vérifier le chargement du Dashboard
3. Vérifier l’absence d’erreur 404 dans la console navigateur

## Parcours 2 : Projet

1. Créer un projet (titre, description, budget prévisionnel, programme)
2. Ouvrir le projet
3. Ajouter une tâche (SMART optionnel)
4. Lier une pièce jointe comme justificatif (si requis)
5. Marquer la tâche comme Réalisé

## Parcours 3 : Comptabilité

1. Accéder au module Comptabilité
2. Créer une écriture
3. Valider l’écriture
4. Verrouiller l’écriture

## Parcours 4 : Droits d’accès

1. Accéder à UserManagement (Droits d’accès)
2. Onglet Permissions Module
3. Sélectionner un utilisateur
4. Cocher plusieurs autres utilisateurs
5. Appliquer les permissions aux sélectionnés

## Parcours 5 : Programme

1. Accéder au module Programme
2. Sélectionner un programme
3. Vérifier l’onglet « Projets liés » avec les projets associés

## Parcours 6 : RH (Phase 3)

1. Accéder au module RH
2. Vérifier les onglets : Congés, Fiche salarié, Fiche poste, Organigramme, Paie, Évaluations, Planning
3. Onglet Paie : créer un bulletin

## Parcours 7 : Logistique

1. Accéder au module Logistique
2. Créer un équipement (si manager)
3. Créer une demande d’équipement
4. Valider / Mettre à disposition (si manager)

## Parcours 8 : Parc automobile

1. Accéder au module Parc automobile
2. Créer un véhicule (si manager)
3. Créer une demande de véhicule
4. Valider / Mettre à disposition (si manager)
